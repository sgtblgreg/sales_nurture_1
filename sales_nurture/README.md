# SegFlow Sales Nurture (после первой встречи)

Задача: для лидов, которые **не купили после 1‑й встречи** (большинство), запускать лёгкий, воспроизводимый follow‑up по методологии SegFlow.

Цель: регулярно давать **маленькие, но ощутимые value‑drops** (PDF‑артефакты) в Telegram, чтобы:
- напоминать о себе и удерживать контекст;
- показывать скорость/глубину методологии;
- создавать “suspense” и FOMO (что рынок движется, а клиент теряет время);
- доводить до полноценного исследования/проекта.

## Что мы знаем после первой встречи (минимум)
Заполняем `lead_brief_template.md`:
- компания/продукт/URL/краткое описание оффера;
- индустрия и география (scope = **продукт × география**);
- список 3–10 известных конкурентов (если есть);
- язык презентации (обычно `ru`) и окно актуальности (например, 12–18 месяцев).

## Что выдаём клиенту (5 касаний / 2 месяца)
Каждое касание — **короткий PDF**, понятный собственнику/CEO:
1) “Источники рынка”: профиль корпуса источников + первые сигналы (почему это важно для точности).
2) “Карта спроса”: 21 сегмент × 52 потребности (scroll) + матрица Segment×Need + сигналы конкурентов/креативы + примеры Need→Functions.
3) “Конкуренты”: карта Competitor×Functions + позиционирование (segments/needs/functions mentions) + вывод по однородности рынка.
4) “GTM‑стратегия (быстрый расчёт)”: 3 связки на короткой выборке + укороченный funnel + базовый sales‑pack для теста.
5) “Мониторинг”: что изменили конкуренты → обновление стратегии и материалов (v2).

## Как пользоваться (для sales)
1) Открой `sales_nurture_react.html`.
2) Нажми **T00** (в таймлайне) или **Контекст** → заполни теги после 1‑й встречи и проверь документ (шаблон).
4) Выбери касание в верхней ленте.
5) Вкладка **Артефакт** → показать/обсудить с лидом (если нужен PDF — обычный print браузера → “Сохранить как PDF”).
6) Вкладка **Сообщение** → скопировать текст и отправить в чат + приложить артефакт.

## Demo PDFs (примеры)
По умолчанию включён DEMO‑режим (demo‑шаблоны). Можно выключить через **Контекст → Очистить**.

> Важно: отчёты “teaser‑формата” **не раскрывают** технические контракты/JSON‑схемы; в UI показываем только методологию.

PS: кнопка **Expert** включает “внутренний” режим (методология + JSON‑блоки в отчётах, если они есть).
